1. Painpoint

Painpoint trung tâm:

Researcher nhận một paper PDF (bài người khác, hoặc bản thảo của chính mình) và muốn biết 2 điều trước khi tin: (1) văn phong/lập luận có vấn đề gì không, (2) các citation có thật không hay đang trích dẫn nguồn ảo. Ngoài ra cần sửa trực tiếp trên văn bản, không chỉ đọc báo cáo riêng rồi tự đi sửa tay từng chỗ.

#PainpointHệ quả thực tếNguồn/khảo sát chứng minh
1 Citation hallucination ở các deep research/LLM-assisted writing tool Reviewer/tác giả không biết citation nào là thật, nào bị bịa — mất uy tín học thuật nếu lọt qua Reference Hallucination in Deep Research Agents (arXiv:2604.03173): benchmark 13 LLM × 40 domain, tỷ lệ hallucinate citation 14%-95%
2 LLM tự review văn phong dễ tự thiên vị hoặc generic không actionable Mô hình review tự động có thể hạ điểm văn bản con người viết có phê bình gắt, hoặc đưa comment chung chung không giúp gì cho việc sửa LLM-REVal (arXiv:2510.12367): LLM reviewer tự bias; MARG baseline cho thấy comment generic chiếm 60% nếu không tách per-section/aspect
3 Tool tự sửa trực tiếp văn bản (không qua approve) gây rủi ro ghi đè sai User mất kiểm soát nội dung gốc, không biết phần nào LLM tự thêm vào, dễ bị "silent overwrite" nội dung quan trọng InkSync (arXiv:2309.15337) "Warn, Verify, Audit" — đề xuất pattern bắt buộc Warn/Verify/Audit khi LLM edit trực tiếp văn bản
4 Annotation neo bằng offset số tuyệt đối trôi vị trí ngay khi văn bản bị sửa ở chỗ khác Comment/highlight chỉ sai 1 ký tự dịch chuyển là trỏ nhầm đoạn văn bản hoàn toàn khác, gây mất tin tưởng vào tool W3C Web Annotation Data Model — TextQuoteSelector, dùng bởi Hypothesis — chuẩn quote+context thay offset số
Quyết định scope (Non-goal): PDF Agent không tự fact-check lại output của chính Research Agent — tránh nghịch lý trust (tool tự sinh nội dung rồi tự nghi ngờ nội dung của mình). Market precedent: Elicit/Consensus (generate) vs Scite (verify) vs Paperpal (polish) luôn tách 3 sản phẩm riêng. Các Non-goal khác: không tự apply suggest/rewrite mà không có Accept/Apply rõ ràng (kể cả confidence cao), không tự sinh citation mới để "fix" warning Not Found, không kiểm tra đạo văn, không dùng ô chat tự do cho thao tác tô-chọn (chỉ 2 action cố định), không tái dựng ảnh/figure thành vector/TikZ editable, không tích hợp OCR/PDF-parsing trả phí (chỉ MinerU self-host).

Lý do: PDF Agent không có bước nào phụ thuộc quyết định user để tiếp tục chạy — P0→P4 chạy 1 lần xong hết, rồi user tương tác với kết quả đã có.

3. Detail Step — công nghệ/API, thuật toán, vấn đề giải quyết

Step P0 — Upload + Format Detection

Công nghệ/APIByte sniffing thuần Python — không cần thư viện ngoài
Thuật toán%PDF → pdf; PK (zip magic) → tex_bundle (main.tex + figures/ + references.bib kiểu export từ Overleaf); \documentclass/\begin{document} trong 2000 byte đầu → tex trần (không có ảnh kèm); còn lại raise UnsupportedFormatError
Vấn đề giải quyếtCần nhận đúng 3 luồng input khác hẳn nhau (PDF quét OCR, .tex Overleaf export có ảnh, .tex viết tay không kèm ảnh) trước khi chọn nhánh parse phù hợp
def detect_format(raw_bytes: bytes) -> Literal["pdf", "tex", "tex_bundle"]:
    if raw_bytes.startswith(b"%PDF"): return "pdf"
    if raw_bytes.startswith(b"PK"):   return "tex_bundle"
    text_head = raw_bytes[:2000].decode("utf-8", errors="ignore")
    if r"\documentclass" in text_head or r"\begin{document}" in text_head:
        return "tex"
    raise UnsupportedFormatError("Không nhận diện được format")

Step P1 — Parse → Structured Document

Công nghệ/API MinerU (self-host CLI, subprocess) cho nhánh PDF; pylatexenc (LatexWalker) cho nhánh .tex/.tex_bundle; zipfile stdlib + path-traversal guard viết tay; pypdf đọc số trang nhanh trước khi quyết định gọi MinerU
Thuật toán Nhánh PDF: mineru -p file.pdf -o out/ -m auto → parse content_list.json (đã có sẵn img_path, image_caption/chart_caption, bbox, page_idx — MinerU tự ghép caption↔figure, tỷ lệ mất caption gần 0) → 1 LLM call temperature=0 dọn reference list thô thành RawCitation[]. Guardrail: reject PDF >PDF_AGENT_MAX_PAGES trước khi gọi MinerU (đọc số trang qua pypdf nhẹ, không cần OCR để biết số trang).
Nhánh .tex_bundle: extract_zip_safe() validate os.path.realpath() mỗi entry nằm trong dest_dir trước extractall() (chống zip-slip) → find_main_tex() heuristic (file có cả \documentclass + \begin{document}) → resolve path ảnh tương đối, copy ảnh tồn tại vào output bundle, đánh dấu missing=True cho ảnh có path trong .tex nhưng không có file trong zip.
Nhánh .tex trần: parse bằng pylatexenc (\section{}, \cite{}+context, \includegraphics{}), ảnh không resolve được → đánh dấu missing=True, KHÔNG raise lỗi cứng (để Step P4 tạo warning missing_asset thay vì chặn cả pipeline).
Vấn đề giải quyếtMỗi nguồn input có rủi ro khác nhau: PDF cần OCR/layout detection chính xác; .zip không tin cậy (path traversal); .tex trần thường thiếu ảnh đi kèm
An ninh bắt buộcMọi .zip upload là input không tin cậy — PHẢI validate path từng entry trước khi extract (OWASP path traversal / "zip slip"). Không dùng zf.extractall() trực tiếp mà không check.
def extract_zip_safe(zip_path: str, dest_dir: str) -> str:
    with zipfile.ZipFile(zip_path) as zf:
        for member in zf.namelist():
            resolved = os.path.realpath(os.path.join(dest_dir, member))
            if not resolved.startswith(os.path.realpath(dest_dir) + os.sep):
                raise SecurityError(f"Zip slip detected: {member}")
        zf.extractall(dest_dir)
    return dest_dir

def find_main_tex(extract_dir: str) -> str:
    for c in glob(f"{extract_dir}/**/*.tex", recursive=True):
        content = read(c)
        if r"\documentclass" in content and r"\begin{document}" in content:
            return c
    raise NoMainTexFoundError()

async def run_mineru(pdf_path: str, output_dir: str) -> dict:
    proc = await asyncio.create_subprocess_exec(
        MINERU_BIN, "-p", pdf_path, "-o", output_dir, "-m", "auto",
        stdout=asyncio.subprocess.PIPE, stderr=asyncio.subprocess.PIPE)
    try:
        await asyncio.wait_for(proc.communicate(), timeout=MINERU_TIMEOUT_S)
    except asyncio.TimeoutError:
        proc.kill()
        raise MinerUTimeoutError(f"MinerU vượt {MINERU_TIMEOUT_S}s")
    return load_json(f"{output_dir}/{stem(pdf_path)}_content_list.json")

Step P3 — Batch Analysis (3 nhánh song song)

P3a Critic Agent Công nghệ: openai/gpt-oss-120b, temperature=0, 1 LLM call/section, asyncio.gather, giới hạn PDF_AGENT_MAX_SECTIONS_CRITIC.
Thuật toán: per-section + 4 aspect cố định (clarity/terminology/flow/redundancy), context isolation (mỗi agent chỉ thấy section của nó), output JSON issue list.
Vấn đề giải quyết: feedback văn phong generic không actionable nếu 1 agent đọc nguyên bài cùng lúc.
Backing: MARG (arXiv:2401.04259) — chia per-section+aspect giảm generic comment 60%→29%; Reviewer2 — khung aspect cố định (gốc novelty/soundness/significance, đổi thành văn phong clarity/terminology/flow/redundancy để khớp Non-goals); LLM-REVal (arXiv:2510.12367) — lý do bắt buộc temperature=0 + prompt conservative, không chấm điểm tổng, chỉ liệt kê issue cụ thể.
P3b Reference Verification Công nghệ: tái dùng semantic_scholar.py/openalex.py/arxiv_search.py của research_agent ở mode "lookup 1 paper" (thêm hàm lookup_by_doi()/lookup_by_id() — bổ sung hàm, không sửa hàm search() cũ), rapidfuzz, LLM judge temperature=PDF_JUDGE_TEMPERATURE(0), giới hạn PDF_AGENT_MAX_CITATIONS_VERIFY=150.
Thuật toán: waterfall — DOI/arXiv ID exact lookup → nếu không có, search song song 3 nguồn (S2/OpenAlex/arXiv, mỗi nguồn top-5) → rapidfuzz multi-field match (title + year±1 + author overlap) → nếu score ≥PDF_AGENT_MATCH_THRESHOLD_HIGH(0.85) → Verified; <PDF_AGENT_MATCH_THRESHOLD_LOW(0.55) → Not Found; vùng xám giữa 2 threshold → LLM judge.
Concurrency (bắt buộc): verify_citations_batch chạy qua asyncio.Semaphore(PDF_AGENT_CITATION_VERIFY_CONCURRENCY=4) — KHÔNG bắn tất cả citation cùng lúc. Mọi lookup dùng chung 1 rate-limiter S2 global (1 req/s); nếu fan-out hàng trăm call đồng thời thì mỗi call xếp hàng lâu hơn citation_lookup_timeout_s → tất cả timeout (CancelledError tại s2_acquire). Bound nhỏ → chỉ vài call S2 tranh nhau 1 lúc → mỗi call xong trong timeout. arXiv chậm/timeout là non-fatal (_safe_call trả None, citation vẫn resolve qua S2/OpenAlex).
Vấn đề giải quyết: phát hiện citation hallucinate/sai metadata mà không cần gọi LLM cho toàn bộ — chỉ vùng xám mới cần LLM, giữ chi phí thấp.
Backing: CheckIfExist (arXiv:2602.15871) — multi-source CrossRef+S2+OpenAlex, multi-field scoring; CiteCheck (arXiv:2605.27700) — waterfall cascade (arXiv→CrossRef→S2→OpenAlex→web) + LLM verifier 0-10 cho case khó, 88.9% accuracy — đã có sẵn 90% hạ tầng từ research_agent Step ①+⑧, chỉ đổi mode "lookup" thay vì "discover".
P3c Link Liveness Check Công nghệ: httpx.AsyncClient().head() song song, timeout PDF_AGENT_LINK_CHECK_TIMEOUT_S(5), không LLM.
Vấn đề giải quyết: raw URL (không phải citation học thuật) có thể đã chết/đổi — chỉ cần check HTTP status, không cần tốn LLM call. MVP chỉ check status code, chưa làm suspicious-domain detection. 
async def verify_citation(c: RawCitation) -> dict:
    if doi := extract_doi(c["raw_text"]):
        if hit := await semantic_scholar.lookup_by_doi(doi):
            return score_match(c, hit)
    candidates = []
    for search_fn in (semantic_scholar.search, openalex.search, arxiv_search.search):
        candidates += await search_fn(c["guessed_title"], n=5)
    best = best_fuzzy_match(c, candidates)
    if best.score >= PDF_AGENT_MATCH_THRESHOLD_HIGH:
        return {"verdict": "Verified", "confidence": best.score, "evidence": best.paper}
    if best.score < PDF_AGENT_MATCH_THRESHOLD_LOW:
        return {"verdict": "Not Found", "confidence": best.score, "evidence": None}
    return await llm_judge_citation_match(c, best)

# Batch: bound concurrency so the shared 1 req/s S2 limiter isn't overwhelmed
async def verify_citations_batch(citations):
    sem = asyncio.Semaphore(PDF_AGENT_CITATION_VERIFY_CONCURRENCY)  # = 4
    async def _one(c):
        async with sem:
            return await verify_citation(c)
    return await asyncio.gather(*(_one(c) for c in citations[:MAX_CITATIONS_VERIFY]),
                                return_exceptions=True)

Step P4 — Build Annotations

Công nghệ/APIW3C Web Annotation Data Model — TextQuoteSelector (exact+prefix+suffix, ~32 ký tự context mỗi bên qua PDF_AGENT_ANCHOR_CONTEXT_CHARS), dùng bởi Hypothesis
Thuật toánbuild_anchor() lưu context trước/sau; refind_anchor() tái định vị bằng regex + disambiguate qua prefix/suffix nếu exact lặp lại nhiều nơi trong văn bản. Không tìm thấy nữa → annotation tự ẩn ở frontend (user đã tự sửa đúng đoạn đó), không báo lỗi
Vấn đề giải quyếtNeo annotation bằng offset số tuyệt đối sẽ trôi sai vị trí ngay khi user sửa 1 ký tự ở đoạn khác — quote+context bền với edit tuỳ ý ở chỗ khác trong văn bản
Ràng buộc cứngtype=warning luôn suggested_fix=None — không có "bản sửa đúng" cho citation giả, Accept sẽ ngầm gợi ý có và gây hiểu lầm. 4 nhóm aspect cho warning: citation_not_found / metadata_mismatch / broken_link / missing_asset; 4 verdict citation: Verified (không tạo warning) / Metadata Mismatch (lệch năm/tác giả) / Not Found (không khớp ở cả 3 nguồn, kèm caveat không tuyệt đối — paper hiếm/sách/non-indexed vẫn có thể không có) / Unreachable (URL trả lỗi HTTP)
def build_anchor(full_text, start, end, context_chars=32) -> TextQuoteSelector:
    return {"exact": full_text[start:end],
            "prefix": full_text[max(0,start-context_chars):start],
            "suffix": full_text[end:end+context_chars]}

def refind_anchor(current_text, anchor) -> int | None:
    candidates = [m.start() for m in re.finditer(re.escape(anchor["exact"]), current_text)]
    if len(candidates) == 1: return candidates[0]
    for pos in candidates:
        if current_text[max(0,pos-32):pos].endswith(anchor["prefix"][-32:]):
            return pos
    return None

Step P5 — Selection-triggered Explain / Rewrite

Công nghệ/APIopenai/gpt-oss-120b — Giải thích temperature=EXPLAIN_TEMPERATURE(0.3) stream; Viết lại temperature=REWRITE_TEMPERATURE(0.5) non-stream, output JSON {old_text, new_text}
Thuật toán2 action cố định (không ô chat tự do, giảm ambiguity scope LLM được làm). Trước khi Apply: validate_rewrite_patch() bắt buộc old_text in current_doc (exact-match) — nếu user đã sửa tay từ lúc tô tới lúc LLM trả lời → reject patch, trả 409, báo "đoạn này đã thay đổi, vui lòng tô lại". Patch ghi vào doc bằng str.replace(old_text, new_text, 1) — chỉ thay occurrence đầu tiên (nếu cần chính xác hơn khi old_text xuất hiện nhiều lần, dùng offset từ anchor đã refind thay vì replace() thuần)
Vấn đề giải quyếtMọi thay đổi văn bản đều phải qua approve rõ ràng (0% tự động ghi đè) — pattern giống IDE diff-apply (Cursor/Copilot/Claude Code Edit tool): constrained diff + exact-match validation
BackingInkSync (arXiv:2309.15337) "Warn, Verify, Audit": Warn↔warning citation, Verify↔evidence từ P3b, Audit↔lịch sử pending_annotations. Usability study cho thấy "more agency", "more accurate, efficient editing" so với chat thường

5. Phương pháp test, đánh giá so sánh

  • E2E nhánh .tex trần: upload → có warning missing_asset nếu thiếu ảnh
  • E2E nhánh .zip: upload project Overleaf mẫu → bundle xuất ra giữ đúng ảnh
  • E2E nhánh .pdf: upload paper arXiv thật → figures + reference verification đúng
  • Test security: .zip chứa zip-slip payload (../../etc/passwd) → reject, không extract ra ngoài dest_dir
  • Test bất biến: mọi Annotation type=warning có suggested_fix is None — assert cứng, không cho lọt qua review code
  • Test anchor: refind_anchor() trả đúng offset sau khi giả lập chèn 50 ký tự vào đoạn text TRƯỚC vị trí anchor
  • Test UI: sửa tay 1 đoạn không liên quan phía trên → annotation phía dưới KHÔNG bị lệch vị trí highlight; sửa đúng đoạn bị flag → annotation đó tự ẩn
  • Test rewrite: giả lập user sửa tay đoạn đã tô NGAY TRƯỚC khi bấm Apply → nhận 409, không ghi đè sai
  • Test concurrency: 2 PATCH annotation cùng lúc trên 1 doc_id — ghi rõ giới hạn SQLite single-writer (MVP single-user, không xử lý multi-user concurrent)
  • Đo cost thực tế 1 document ~60 citation, ~15 section — so với ước tính SPEC (~$0.003/document nhánh .pdf)
  • Đo latency P0→P4 cho PDF 60 trang (gồm MinerU) — guardrail timeout có kích hoạt đúng không
